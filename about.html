Cloudq Scheduler

A scalable event manager

What is it?

CS is a cloudq worker that consumes jobs that can be scheduled and triggered based on that schedule

{ job:
  { 
    klass: 'Add',
    args: [{
       name: 'backup-database',
       schedule: '00 30 11 * * 2-6',
       queue: 'backup',
       job: {
         klass: 'Backup',
         args: ['full']
       }
    }]
  }
}

When I publish a job to the scheduler queue.

I send the job I want to schedule

The name of the queue to publish to

A unique name for the scheduled job

and this '00 30 11 * * 2-6'

What?

Its CRON!

Every M - F at 11:30 AM

Run my full backup

Each Schedule must have unique name

Which is easy to remove

{ job:
  { 
    klass: 'Remove',
    args: [{
      name: 'backup-database',
    }]
  }
}

Simply use any cloudq client to publish to the "Scheduler" queue

What is CRON?

Six nodes to represent re-occuring or specific time

First Node is 0-60 seconds

00 * * * * *
Every Minute at 00 second

*/5 * * * * *
Every 5 seconds

10-20 * * * * *
Run each second between second 10 and 20 of a whole minute

second minute hour day_of_month month day_of_week
